# instruction-facture-reparation.md

## 🎯 Objectif

Créer une nouvelle page web locale nommée `facture-reparation.html`, permettant de générer une facture de réparation véhicule (atelier).

Cette page doit s’intégrer dans le mini logiciel existant et reprendre :

* le design du générateur actuel
* la logique formulaire → génération PDF
* la structure A4 professionnelle

Le rendu final doit être cohérent avec :

* facture vente
* facture acompte
* générateur dossier client

---

## 🧱 Structure générale

La page doit contenir :

1. Une vue formulaire (saisie)
2. Une vue impression (format A4)
3. Un bouton “Générer facture”
4. Un bouton “Retour modification”
5. Un bouton “Imprimer / PDF”

---

## 📄 Sections du formulaire

### 1. Informations facture

Champs :

* `f_num`
* `f_date`
* `f_pay` (Chèque / Virement)
* `f_ref` (référence dossier ou ordre de réparation)

---

### 2. Client

Champs :

* `c_nom`
* `c_prenom`
* `c_addr`
* `c_city`
* `c_tel`

---

### 3. Véhicule

Champs :

* `v_marque`
* `v_modele`
* `v_immat`
* `v_vin`
* `v_km`
* `v_mec`
* `v_energie`

---

### 4. Détail des prestations (PARTIE PRINCIPALE)

Créer un système de lignes dynamiques.

Chaque ligne contient :

* `designation` (ex : vidange, plaquettes, diagnostic)
* `type` (pièce / main-d’œuvre / forfait)
* `prix_ht`
* `tva`
* `prix_ttc`

Fonctionnalités :

* bouton “+ ajouter ligne”
* bouton supprimer ligne
* recalcul automatique TVA
* recalcul dynamique des totaux

---

### 5. Totaux

Champs calculés :

* `total_ht`
* `total_tva`
* `total_ttc`
* `montant_paye`
* `reste_du`

Règles :

* total HT = somme des HT
* TVA = somme TVA
* TTC = somme TTC
* reste = TTC - montant payé

---

### 6. Observations / Commentaires atelier (IMPORTANT)

Créer une zone texte :

Champ :

* `t_obs`

Objectif :
Permettre au chef d’atelier ou technicien de rédiger :

* détail des réparations effectuées
* remarques techniques
* anomalies détectées
* recommandations client

Contraintes :

* textarea large
* affichage clair dans le PDF
* bloc visuellement séparé

---
